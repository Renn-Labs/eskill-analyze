# Quality Principles

These override everything else in analysis and execution:

1. **Selective, not exhaustive** — Only apply frameworks that are relevant to THIS specific analysis. Skip the rest.
2. **Actionable over comprehensive** — Every recommendation must be something someone can go build TODAY. No vague "improve the experience" items.
3. **Fewer, better priorities** — Cap the action list at 10-15 items max. If you have 40, you haven't prioritized.
4. **Name the simplest version** — For each action, describe the MVP that delivers 80% of the value. Not the dream version.
5. **Cut the filler** — No restating the obvious. No padding sections to look thorough. Empty sections get deleted, not filled with platitudes.
6. **Quantify or don't claim** — "Improves retention" is slop. "Reduces churn at day-7 by giving users a reason to return" is useful.
